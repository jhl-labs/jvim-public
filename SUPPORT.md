# Support

Need help with `jvim`? Here's where to go.

## First: check the built-in help

Press **F1** inside `jvim` for the keymap. Most "how do I…" questions are
answered there.

## Documentation

- [`README.md`](./README.md) — install and quick start
- [`docs/install.md`](./docs/install.md) — detailed install paths and
  troubleshooting
- [`docs/faq.md`](./docs/faq.md) — frequently asked questions
- [`CHANGELOG.md`](./CHANGELOG.md) — what changed when

## Ask a question

Open a [Question](https://github.com/jhl-labs/jvim-public/issues/new?template=question.yml)
issue.

## Report a bug

Open a [Bug report](https://github.com/jhl-labs/jvim-public/issues/new?template=bug_report.yml).

## Report a security vulnerability

**Do not open a public issue.** See [`SECURITY.md`](./SECURITY.md).

## Commercial support

Not currently offered. If your organization has specific needs, file a
[Question](https://github.com/jhl-labs/jvim-public/issues/new?template=question.yml)
issue and we'll follow up.

## Response times

This is a small-team project. Best-effort response within a few business
days. Security reports are prioritized — see [`SECURITY.md`](./SECURITY.md)
for formal SLAs.
