Notepad-ergonomic, vim-fast, AI-native.

마크다운 노트를 위한 터미널 편집기. wikilink·tag·git·semantic search·AI overlay까지 단일 바이너리에. 설치 1줄, npm 권장.

$ npm install -g @jhl_labs/jvim

기능

마크다운 워크스페이스에 필요한 것을 한 바이너리에. node_modules도, Node 런타임도 필요 없습니다.

md 마크다운 우선 편집

헤딩·강조·리스트·코드펜스·테이블·인라인 링크에 라이브 스타일링. 페이지가 아닌 워크스페이스 단위.

[[ ]] Wikilink + Backlink

[[file]] 자동완성, outbound 오버레이, backlink 그래프. Obsidian 호환 데이터 모델.

# 태그 브라우저

#tag 자동완성과 F7 태그 브라우저. 태그-검색 핸드오프로 한 번에 vault 전체 탐색.

🔎 풀텍스트 + 시맨틱 검색

SQLite FTS5 기본, 옵션으로 sqlite-vec 임베딩. 대소문자 토글, vault-wide 검색.

⌘ Notepad 단축키

Ctrl+C/X/V/A/Z/Y/S/O/N + Shift+Arrow 선택. vim 스타일 트리(j/k/g/G/n/r/d)도 함께.

AI AI 오버레이

인라인 자동완성, 빠른 프롬프트, 모드별 컨텍스트 (Ask / Insert / Rewrite). Local LLM도 지원.

git Git 통합

커밋 로그 뷰어(F5), 헤더의 +N/−M diff 요약과 jvim diff CLI. vault-aware 인덱스가 git checkout 후에도 유지됨.

📦 단일 바이너리

플랫폼당 한 파일. Node 런타임도 node_modules도 필요 없음. npm 또는 GitHub Releases.

설치

지원 플랫폼: linux-x64, linux-arm64. macOS / Windows 예정.

npm 권장

플랫폼 바이너리는 optionalDependencies로 자동 선택됩니다. CLI 명령은 그대로 jvim. 

# 신규 설치
npm install -g @jhl_labs/jvim

# 업그레이드
npm install -g @jhl_labs/jvim@latest --include=optional

jvim --version

GitHub Release 에어갭

스탠드얼론 tarball + SHA-256 체크섬. 사내망·CI 캐시·오프라인 환경에 적합.

# {{ site.version }} / linux-x64 예시
curl -LO https://github.com/jhl-labs/jvim-public/releases/download/v{{ site.version }}/jvim-{{ site.version }}-linux-x64.tar.gz
curl -LO https://github.com/jhl-labs/jvim-public/releases/download/v{{ site.version }}/jvim-{{ site.version }}-linux-x64.tar.gz.sha256
sha256sum -c jvim-{{ site.version }}-linux-x64.tar.gz.sha256
tar -xzf jvim-{{ site.version }}-linux-x64.tar.gz
sudo mv jvim /usr/local/bin/

핵심 사용법

설치 후 바로 써볼 6가지 핵심 워크플로. 모든 스크린샷은 실제 jvim 실행 결과입니다.

편집 기초

즉시 편집 가능한 TUI

파일 한 개든 vault든 열자마자 편집 가능. 파일 트리는 왼쪽, 에디터는 오른쪽. 상단 상태바에 파일명·EOL·저장 상태가 실시간으로 표시됩니다. Tab으로 들여쓰기, Ctrl+E로 트리↔에디터 포커스 전환.

  1. jvim notes.md 또는 jvim my-vault/로 시작
  2. 편집하면 상태바에 ● 표시 (unsaved)
  3. Ctrl+S로 저장 — 원자적 쓰기로 파일 손실 위험 없음
  4. 새 파일은 jvim new.md만 하면 자동 생성 (vi/notepad 시맨틱)
빠른 이동

커맨드 팔레트 + 파일 팔레트

Ctrl+F4(또는 F4)는 모든 명령을 퍼지 검색으로 실행. Ctrl+O는 vault 내 파일을 순간 열기. Ctrl+Tab으로 열린 버퍼 사이클.

  1. F4 — 커맨드 팔레트 (save, close, toggle-line-numbers, …)
  2. Ctrl+O — 파일 팔레트 (vault 내 전체 파일 fuzzy)
  3. F2 — 현재 문서 아웃라인 (heading 점프)
  4. Ctrl+Tab/Ctrl+Shift+Tab — 열린 버퍼 사이클
검색

문서 내 + vault 전체 검색

Ctrl+F는 현재 문서 내 검색 (Tab으로 대소문자 토글). Shift+F4는 vault 전체 풀텍스트 검색 (FTS5 + 선택적 시맨틱). 태그·wikilink 토큰도 쿼리로 그대로.

  1. Ctrl+F — 문서 내 검색 · Enter 다음, Shift+Enter 이전
  2. Ctrl+T — replace-all
  3. Shift+F4 — vault 전체 검색 (제목·본문·태그)
  4. #tag, [[link]] 쿼리에 그대로 쓸 수 있음
AI 활용

Ask / Insert / Rewrite — 세 가지 모드

Ctrl+/ 또는 F6으로 AI 오버레이 열기. 현재 선택·섹션·빈 컨텍스트 중 하나를 자동으로 잡고, 세 모드 중 하나로 행동. OpenAI 호환·Anthropic·로컬 LLM 모두 지원.

  1. Ctrl+/ — 오버레이 토글 (또는 F6)
  2. 모드 선택: Ask (답만 오버레이로), Insert (커서에 삽입), Rewrite (선택 영역 교체)
  3. 선택이 있으면 컨텍스트로 자동 사용; 없으면 line-window / 섹션 fallback
  4. F9로 인라인 자동완성 토글 (ghost text), →로 수락
지식 그래프

Wikilink · Backlink · Tag

Obsidian 호환 데이터 모델 — [[file]] 자동완성, 전방/후방 연결 그래프, 태그 브라우저까지. 어떤 문서에서든 그래프를 따라 이동 가능.

  1. [[ 입력 → 실시간 파일 자동완성
  2. F8 — 현재 문서의 outbound 링크 오버레이
  3. Ctrl+B — backlink 그래프 (이 문서를 참조하는 문서들)
  4. F7 — tag 브라우저 · Enter로 tag:<name> 검색 핸드오프
Git 워크플로

커밋 로그 · diff 요약 · CLI

vault는 기본적으로 git 저장소. 상태바에 +N/−M 실시간 diff 요약. F5로 commit log를 열고, jvim diff CLI로 review 가능. AI 편집은 reviewable diff로 떨어집니다.

  1. F5 — 커밋 로그 오버레이 (작성자·SHA·메시지)
  2. 상단 헤더의 +12 −4 — 현재 문서의 미커밋 diff
  3. jvim diff [file] — CLI에서 pager 스타일 diff
  4. vault-aware 인덱스는 git checkout 이후에도 유지
