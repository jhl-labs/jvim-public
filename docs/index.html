마크다운 노트를 위한 터미널 편집기. wikilink·tag·git·semantic search·AI overlay까지 단일 바이너리에. 설치 1줄, npm 권장.

기능

마크다운 워크스페이스에 필요한 것을 한 바이너리에. node_modules도, Node 런타임도 필요 없습니다.

md 마크다운 우선 편집

헤딩·강조·리스트·코드펜스·테이블·인라인 링크에 라이브 스타일링. 페이지가 아닌 워크스페이스 단위.

[[ ]] Wikilink + Backlink

[[file]] 자동완성, outbound 오버레이, backlink 그래프. Obsidian 호환 데이터 모델.

# 태그 브라우저

#tag 자동완성과 F7 태그 브라우저. 태그-검색 핸드오프로 한 번에 vault 전체 탐색.

🔎 풀텍스트 + 시맨틱 검색

SQLite FTS5 기본, 옵션으로 sqlite-vec 임베딩. 대소문자 토글, vault-wide 검색.

⌘ Notepad 단축키

Ctrl+C/X/V/A/Z/Y/S/O/N + Shift+Arrow 선택. vim 스타일 트리(j/k/g/G/n/r/d)도 함께.

AI AI 오버레이

인라인 자동완성, 빠른 프롬프트, 모드별 컨텍스트 (Ask / Insert / Rewrite). Local LLM도 지원.

git Git 통합

커밋 로그 뷰어(F5), 헤더의 +N/−M diff 요약과 jvim diff CLI. vault-aware 인덱스가 git checkout 후에도 유지됨.

# 업그레이드
npm install -g @jhl_labs/jvim@latest --include=optional

jvim --version

GitHub Release 에어갭

스탠드얼론 tarball + SHA-256 체크섬. 사내망·CI 캐시·오프라인 환경에 적합.

# {{ site.version }} / linux-x64 예시
curl -LO https://github.com/jhl-labs/jvim-public/releases/download/v{{ site.version }}/jvim-{{ site.version }}-linux-x64.tar.gz

tar -xzf jvim-{{ site.version }}-linux-x64.tar.gz
sudo mv jvim /usr/local/bin/

상세 사용 가이드

jvim의 모든 핵심 기능을 스크린샷과 함께 하나씩 안내합니다. 도움말(F1)에서 언급되는 모든 기능이 여기에 포함됩니다.

편집 기초

즉시 편집 가능한 TUI

파일 한 개든 vault 폴더든 열자마자 편집할 수 있습니다. 왼쪽에 파일 트리, 오른쪽에 에디터. 상단 상태바에 파일명·EOL·저장 상태·git diff 요약이 실시간으로 표시됩니다. 빈 문서의 EOF 영역은 vim 스타일 ~로 채워 뷰포트 경계를 명확히 합니다.

  1. jvim notes.md 또는 jvim my-vault/로 시작
  2. 편집하면 상태바에 ● 표시 (unsaved)
  3. Ctrl+S로 저장 — 원자적 쓰기 + fsync로 파일 손실 방지
  4. 존재하지 않는 파일 경로도 OK: jvim new.md하면 빈 버퍼 생성 (vi/notepad 시맨틱)
  5. Ctrl+Z/Ctrl+Y로 undo/redo · Ctrl+A 전체 선택
파일 트리

vim 스타일 트리 + 실시간 필터

파일 트리에서 j/k로 이동, Enter로 열기. 글자를 입력하면 실시간 type-to-filter로 파일을 순식간에 찾습니다. n/r/d로 새 파일·이름변경·삭제까지 트리 안에서 완결. 깊이 중첩된 디렉토리도 자동 접기/펼치기.

  1. Ctrl+E 또는 Tab(트리 포커스) — 에디터 ↔ 트리 전환
  2. j/k — 위·아래 이동 · g/G — 맨 위/아래
  3. 아무 글자 입력 → 실시간 필터 (위 스크린샷처럼 즉시 좁혀짐)
  4. n 새 파일 · r 이름 변경 · d 삭제 (확인 대화상자)
  5. Enter로 열기 · q로 에디터 복귀
빠른 이동

커맨드 팔레트 · 파일 팔레트 · 아웃라인

F4로 모든 명령을 fuzzy 검색 실행. Ctrl+O로 vault 내 아무 파일을 즉시 열기. F2로 현재 문서의 heading 아웃라인을 열어 긴 문서를 빠르게 탐색. Shift+F3으로 줄 번호 점프.

  1. F4 / Ctrl+P — 커맨드 팔레트 (save, close, toggle-line-numbers, …)
  2. Ctrl+O — 파일 팔레트 (vault 전체 fuzzy)
  3. F2 — 아웃라인 오버레이: heading으로 점프
  4. Shift+F3 — Go to line 대화상자 (줄 번호 입력 → 점프)
  5. Ctrl+Tab/Ctrl+Shift+Tab — 열린 버퍼 사이클
아웃라인

F2 — 문서 구조를 한눈에

현재 문서의 마크다운 heading (# ~ ######)을 깊이별로 계층화하여 보여줍니다. Enter로 해당 heading 위치로 즉시 점프. 긴 문서에서 원하는 섹션을 찾는 가장 빠른 방법.

  1. F2 — 아웃라인 열기
  2. ↑/↓로 heading 선택
  3. Enter — 해당 라인으로 커서 점프 + 오버레이 닫힘
  4. Esc — 점프 없이 닫기
검색 · 치환

Ctrl+F 검색 + Ctrl+T Replace-all

문서 내 검색은 Ctrl+F. 실시간 하이라이트와 함께 Enter/Shift+Enter으로 다음/이전 매치를 순회합니다. Tab으로 대소문자 구분을 토글(상태바에 Aa 뱃지 표시). Ctrl+T는 전체 치환 대화상자.

  1. Ctrl+F — 검색 오버레이 열기
  2. 검색어 입력 → 실시간 하이라이트 (매치 수 표시)
  3. Tab — 대소문자 구분 토글 (Aa 뱃지 on/off)
  4. Enter 다음 매치 · Shift+Enter 이전 매치
  5. Ctrl+T — replace-all: 검색어 + 교체어 입력 후 한 번에 치환
Vault 검색

Shift+F4 — 풀텍스트 + 시맨틱 검색

vault 내 모든 파일의 제목·본문·태그를 SQLite FTS5로 풀텍스트 검색. 선택적으로 sqlite-vec 임베딩 기반 시맨틱 검색까지. 쿼리에 #tag, [[link]] 토큰을 그대로 사용 가능. 알 수 없는 접두사나 닫히지 않은 따옴표에는 경고 표시.

  1. Shift+F4 — vault 전체 검색 오버레이
  2. 쿼리 입력 → 실시간 결과 (파일명·매치 라인 미리보기)
  3. #travel 같은 태그, [[chunker]] 같은 wikilink도 쿼리로 사용
  4. Enter — 선택한 결과 파일 열기
  5. 시맨틱 검색은 config.toml에서 [ai.embeddings] 설정 시 활성화
태그 시스템

#tag 자동완성 + F7 태그 브라우저

에디터에서 #를 입력하면 vault 전체의 태그를 실시간 자동완성. F7 태그 브라우저는 모든 태그를 접두사 필터와 함께 나열하며, Enter로 선택하면 tag:<name> vault 검색으로 핸드오프됩니다.

  1. 에디터에서 # 입력 → 실시간 태그 자동완성 팝업
  2. ↑/↓로 선택, Enter로 삽입
  3. F7 — 태그 브라우저: vault 전체 태그 일람
  4. 브라우저에서 Enter → tag:<name>으로 vault 검색 핸드오프
Wikilink

[[파일]] 자동완성 + F8 Outbound 링크

에디터에서 [[를 입력하면 vault 내 파일을 실시간 fuzzy 자동완성. 완성된 wikilink는 클릭 없이 문서 간 연결을 형성합니다. F8은 현재 문서에서 나가는 모든 wikilink를 오버레이로 보여주며, Enter로 대상 파일 열기.

  1. 에디터에서 [[ 입력 → vault 파일 자동완성
  2. ↑/↓로 선택, Tab 또는 Enter로 완성
  3. F8 — outbound 링크 오버레이: 이 문서가 참조하는 문서들
  4. Enter — 선택한 링크 대상 파일로 이동
Backlink 그래프

Ctrl+B — 이 문서를 참조하는 모든 문서

Obsidian 스타일 backlink 탐색. Ctrl+B로 현재 문서를 [[wikilink]]로 참조하는 모든 문서를 그래프 형태로 조회합니다. [/]로 이웃 문서를 호핑하며, 하이라이트 경로가 유지되어 어디서 왔는지 추적 가능.

  1. Ctrl+B — backlink 그래프 열기
  2. ↑/↓ — 백링크 리스트 탐색
  3. [/] — 이웃 문서로 hop (경로 유지)
  4. Enter — 선택한 문서를 에디터에서 열기
  5. Esc — 그래프 닫기
AI 오버레이

Alt+/ — Ask · Insert · Rewrite 세 가지 모드

Alt+/ (또는 Ctrl+/ / F6)로 AI 오버레이를 열면 세 가지 모드 중 하나를 선택합니다. Ask는 오버레이에서 질의응답, Insert는 커서 위치에 생성 텍스트 삽입, Rewrite는 선택 영역을 AI가 교체합니다. 선택이 있으면 자동 컨텍스트, 없으면 섹션/line-window 폴백. OpenAI 호환·Anthropic·로컬 LLM 모두 지원. 11개 빌트인 컨텍스트 액션(요약, 번역, 코드 리뷰, 리팩토링 등) 제공.

  1. Alt+/ — AI 컨텍스트 메뉴 (커서 위치에 표시)
  2. 모드 선택: Ask / Insert / Rewrite
  3. 선택 영역이 있으면 컨텍스트로 자동 전달
  4. AI 응답은 스트리밍 — 상태바에 진행 인디케이터 표시
  5. Esc로 취소 · Rewrite 결과가 마음에 안 들면 Ctrl+Z
AI 자동완성

Copilot 스타일 인라인 ghost text

Alt+I (또는 F9)로 인라인 자동완성을 토글합니다. 활성화되면 타이핑을 멈출 때 AI가 다음 내용을 ghost text로 제안. Tab으로 수락, 다른 키를 누르면 무시됩니다. 파일 헤더·현재 스코프·스타일을 자동으로 읽어 context-aware 자동완성. 상태바의 AI 뱃지로 on/off 상태와 에러 원인을 즉시 확인.

  1. Alt+I — 인라인 자동완성 토글 (F9도 동작)
  2. 타이핑 → 잠깐 멈추면 ghost text 표시 (흐릿한 텍스트)
  3. Tab — ghost text 수락 (VSCode 컨벤션)
  4. 다른 키 입력 시 제안 자동 해제
  5. providers.toml에서 inline_model 설정으로 빠른 모델 분리 가능
구문 강조

Shiki 기반 100+ 언어 코드 하이라이팅

마크다운의 코드펜스 (```js ... ```) 안 코드가 Shiki 엔진으로 구문 강조됩니다. JavaScript, TypeScript, Python, Go, Rust, Bash 등 100개 이상 언어를 지원. 에디터 안에서 코드 블록이 읽기 좋게 색상 구분되어 기술 문서 작성에 최적.

  1. 마크다운에서 ```언어명으로 코드펜스 시작
  2. 코드가 자동으로 구문 강조됨 (터미널 256색/truecolor)
  3. 지원 언어: js, ts, py, go, rust, bash, json, yaml, sql, html, css …
  4. 강조는 실시간 — 코드를 편집하면 즉시 갱신
Git 통합

F5 커밋 로그 · 헤더 diff · jvim diff CLI

vault는 기본적으로 git 저장소. 상태바 헤더에 +N/−M으로 현재 파일의 미커밋 변경 라인 수를 실시간 표시. F5로 커밋 로그 오버레이(작성자·SHA·메시지), jvim diff로 CLI에서 pager 스타일 리뷰 가능. AI 편집 결과도 diff로 확인 가능.

  1. F5 / Ctrl+L — 커밋 로그 오버레이
  2. 상단 헤더의 +12 −4 — 현재 파일의 미커밋 diff 요약
  3. jvim diff [file] — CLI pager 스타일 diff
  4. vault-aware 인덱스는 git checkout 후에도 유지
  5. Ctrl+R — 전체 재인덱싱 (필요 시 수동 트리거)
설정

F10 — config.toml 편집 + 즉시 반영

F10 (또는 Ctrl+K)으로 설정 오버레이를 열어 config.toml을 직접 편집합니다. AI 프로바이더, 줄 번호, 탭 크기, scroll-off, 업데이트 체크 등을 설정. 저장하면 재시작 없이 즉시 반영됩니다. 잘못된 TOML은 에러 토스트로 알림.

  1. F10 / Ctrl+K — 설정 오버레이 열기
  2. [editor] — tab_size, scroll_off, line_numbers
  3. [ai] — provider, model, inline_model, quick_prompts
  4. [updates] — check_on_start (npm 업데이트 체크 on/off)
  5. Ctrl+S 저장 시 즉시 반영 · 파싱 에러 시 이전 설정 유지
