## Why jvim?

<div class="why-grid">
  <div class="why-card">
    <span class="why-tag">UX</span>
    <h3>No mode switching</h3>
    <p>Insert mode is the default — like Notepad. vim users keep <code>j</code>/<code>k</code>/<code>g</code>/<code>G</code>/<code>n</code>/<code>r</code>/<code>d</code> in the tree.</p>
  </div>
  <div class="why-card">
    <span class="why-tag">Vault</span>
    <h3>Folders, not pages</h3>
    <p>Open a folder, navigate with the file tree, link with <code>[[wikilinks]]</code>, tag with <code>#topics</code>.</p>
  </div>
  <div class="why-card">
    <span class="why-tag">AI</span>
    <h3>AI stays optional</h3>
    <p>When configured, Ask mode can search the vault first and show source notes. Without a provider, jvim remains a local editor.</p>
  </div>
  <div class="why-card">
    <span class="why-tag">Ship</span>
    <h3>Single binary</h3>
    <p>Download, run. No node_modules, no Node runtime, no Electron. Linux x64 / arm64.</p>
  </div>
</div>
