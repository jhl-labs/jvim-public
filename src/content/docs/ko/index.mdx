## 왜 jvim?

<div class="why-grid">
  <div class="why-card">
    <span class="why-tag">UX</span>
    <h3>모드 전환 없음</h3>
    <p>기본이 Insert 모드 — Notepad처럼. vim 사용자는 트리에서 <code>j</code>/<code>k</code>/<code>g</code>/<code>G</code>/<code>n</code>/<code>r</code>/<code>d</code> 그대로.</p>
  </div>
  <div class="why-card">
    <span class="why-tag">Vault</span>
    <h3>페이지가 아닌 폴더</h3>
    <p>폴더를 열고, 파일 트리로 탐색하고, <code>[[wikilinks]]</code>로 연결하고, <code>#태그</code>로 분류합니다.</p>
  </div>
  <div class="why-card">
    <span class="why-tag">AI</span>
    <h3>AI는 선택 사항</h3>
    <p>설정한 경우 Ask 모드가 vault를 먼저 검색하고 출처 노트를 보여줄 수 있습니다. 설정하지 않아도 로컬 편집기로 동작합니다.</p>
  </div>
  <div class="why-card">
    <span class="why-tag">Ship</span>
    <h3>단일 바이너리</h3>
    <p>다운로드해서 실행. node_modules·Node 런타임·Electron 없음. Linux x64 / arm64.</p>
  </div>
</div>
