jvim의 Tier 2 시맨틱 인덱스는 `sqlite-vec`으로 벡터 임베딩을 생성하고 vault 옆에 로컬로 저장합니다. 풀텍스트 검색과 달리, 시맨틱 인덱스는 키워드를 공유하지 않더라도 AI Ask 모드가 개념적으로 관련된 노트를 찾을 수 있게 합니다. 인덱싱할 파일을 직접 선택하며 — jvim이 vault 전체를 자동으로 임베딩하지 않습니다.

## 스코프 선택기 열기

`Shift+F5`를 눌러 스코프 선택 대화상자를 엽니다. 대화상자는 스코프 옵션을 제시하고 작업을 시작하기 전에 선택을 기다립니다.

## 스코프 옵션

| 스코프 | 인덱싱 대상 |
|---|---|
| **현재 파일** | 에디터에서 열려 있는 파일만. 빠르며 단일 노트의 빠른 AI 지원 검토에 유용합니다. |
| **열린 문서** | 현재 탭에 열려 있는 파일들. 작은 노트 묶음을 함께 작업할 때 유용합니다. |
| **Vault** | vault의 모든 파일. 가장 완전한 인덱스; 대규모 vault에서 시간이 걸릴 수 있으므로 시작 전에 확인 대화상자를 표시합니다. |
| **선택한 문서** | 선택한 문서 규칙에 매칭되는 파일. vault 전체를 인덱싱하지 않고 반복 가능한 부분집합을 만들 때 유용합니다. |

스코프를 선택하면 jvim이 백그라운드에서 임베딩 인덱스를 구축(또는 증분 업데이트)합니다. 구축이 실행되는 동안 상태바에 진행 표시기가 나타납니다.

## 대규모 스코프 확인

**Vault**를 선택하고 vault에 많은 파일이 있으면 jvim은 임베딩할 파일 수를 추정하는 확인 대화상자를 표시합니다. 이를 통해 시작 전에 스코프와 예상 처리 시간을 검토할 수 있습니다. `Enter`로 진행하거나 `Esc`로 취소합니다.

## 선택한 문서

**선택한 문서** 스코프는 스코프 대화상자의 선택 문서 규칙 관리자를 사용합니다. 파일이나 폴더 규칙을 추가해 vault 전체를 임베딩하지 않고 반복 가능한 인덱싱 부분집합을 만들 수 있습니다.

## AI Ask 모드와 RAG

인덱스가 구축되면 AI Ask 모드(`F6 → Ask`)가 이를 검색 소스로 사용할 수 있습니다. 질문을 하면 jvim이 쿼리를 임베딩하고 인덱스에서 의미적으로 가장 관련 있는 청크를 컨텍스트로 포함합니다. RAG 컨텍스트의 품질은 선택한 스코프에 직접 달려 있습니다: 더 광범위하고 최신 인덱스일수록 더 관련성 높은 결과를 제공합니다.

AI 컨텍스트에서 사용 가능하도록 하고 싶은 노트를 추가할 때마다 `Shift+F5`를 다시 실행하세요.
